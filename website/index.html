how it works
01

Identity

Every agent request carries a did:key DID and an Ed25519 signature. Sites know exactly who is asking and can block bad actors by DID.

02

Policy

Sites publish /.well-known/tollway.json — a machine-readable file declaring allowed scopes, pricing, attribution requirements, and rate limits.

03

Payments

HTTP 402 + x402. When content requires payment, the server returns a price and wallet address. The agent pays in USDC on Base and retries.

The request lifecycle

1

Agent fetches the site's policy

Before the first request, the agent fetches /.well-known/tollway.json. This tells it what scopes are allowed, whether payment is required, and attribution rules. The policy is cached for up to an hour.

2

Agent sends a signed request

The agent attaches X-Tollway-* headers — its DID, purpose, scope, a unique nonce, and an Ed25519 signature over the canonical request string. The server can verify the request was made by who it claims.

3

Server validates and enforces policy

The server verifies the signature, checks the nonce (replay protection), enforces the allowed/prohibited action list, and applies rate limits. Non-Tollway requests pass through unchanged.

4

Payment (if required)

If the scope requires payment, the server returns 402 with a price and USDC wallet address. The agent sends the payment on-chain via Base and retries with the transaction hash. The server verifies on-chain and serves the content.

// npm install @tollway/server import { tollwayMiddleware } from '@tollway/server'; app.use(tollwayMiddleware({ policy: { trainingAllowed: false, attributionRequired: true, freeRequestsPerDay: 100, paymentRequiredActions: ['train'], pricing: [{ action: 'train', price: '0.05' }], }, paymentAddress: process.env.PAYMENT_ADDRESS, }));
// middleware.ts — npm install @tollway/server import { createNextjsMiddleware } from '@tollway/server'; export default createNextjsMiddleware({ policy: { trainingAllowed: false, freeRequestsPerDay: 500, paymentRequiredActions: ['train'], }, paymentAddress: process.env.PAYMENT_ADDRESS, }); export const config = { matcher: ['/((?!_next).*)' ] };
# pip install tollway-server from tollway import TollwayFastAPI, ServerPolicy app.add_middleware( TollwayFastAPI, policy=ServerPolicy( training_allowed=False, attribution_required=True, free_requests_per_day=100, payment_required_actions=["train"], pricing=[{"action": "train", "price": "0.05"}], ), payment_address=os.environ["PAYMENT_ADDRESS"], )
// npm install @tollway/client @tollway/payments import { createAgent } from '@tollway/client'; import { createPaymentHandler } from '@tollway/payments'; const agent = createAgent({ did: process.env.AGENT_DID, privateKey: process.env.AGENT_KEY, purpose: 'Research', scope: 'read', onPaymentRequired: createPaymentHandler({ walletPrivateKey: process.env.WALLET_KEY, maxPriceUsdc: '0.10', }), }); const result = await agent.fetch('https://example.com/article'); console.log(result.paid, result.cost, result.data);

The ecosystem

@tollway/client
Drop-in fetch replacement with identity headers and payment flows
TypeScript · npm
@tollway/server
Middleware for Express and Next.js. Serves policy, validates identity, handles 402.
TypeScript · npm

Generate DIDs, fetch URLs with identity, inspect site policies
TypeScript · npm
